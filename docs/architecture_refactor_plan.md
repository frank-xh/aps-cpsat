# 冷轧排程架构重构计划（CP-SAT 一体化）

## 当前已完成
1. 分层目录与门面流程：
   - `preprocess -> transition -> model -> decode -> validate`
2. `transition` 层已实现 `(i,j,line)` 稀疏转移模板预计算（含最小桥接块数与成本）。
3. `model` 层已接收模板包，当前仍复用旧求解内核。

## 下一步（按顺序）
1. 在 `model/master.py` 引入“主模型变量骨架”：
   - `u[i]`: 未排
   - `y[i,l,p]`: 订单-线别-辊期槽位
   - `x[l,p,i,j,t]`: 弧模板选择
2. 把“解后删单/解后断期”前移为模型内决策：
   - 未排由 `u[i]` 决定并计罚
   - 弧模板本身保证相邻合法
3. 把“辊期吨位上下限/目标吨位偏差”放进主模型。
4. 解后层仅做模板展开和导出，不再做可行性补救。

## 关键原则
1. 一体化的是业务决策，不是一锅端的显式位置模型。
2. 虚拟板坯以“弧模板属性”建模，不作为海量显式节点。
3. 任何硬约束尽量通过“候选弧过滤 + 模型约束”实现，避免解后删除。
