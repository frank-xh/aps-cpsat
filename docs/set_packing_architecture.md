# Set Packing (Macro-Block Assembly) Architecture

## Overview

The Set Packing architecture is a new approach to the cold rolling scheduling problem that abandons the "Master assigns un-sequenced orders -> Local Router fixes them" architecture in favor of a cleaner two-phase approach:

1. **Phase 1 (Block Generation)**: Generate a pool of fully-valid candidate macro-blocks
2. **Phase 2 (Set Packing)**: Select a non-overlapping subset that maximizes total scheduled tonnage

## Architecture Comparison

### Old Architecture (Master + Local Router)
```
Orders → Master CP-SAT (line/slot assignment) → Local Router (sequencing) → Plan
         ↑
         ↓
    May use illegal fallback edges → Hard constraint violations
```

### New Architecture (Set Packing)
```
Orders → Block Generator (DFS graph walk) → Candidate Blocks
              ↓
        Set Packing CP-SAT → Selected Blocks → Direct Unpack → Plan
              ↓
        100% valid template edges (by construction)
```

## Key Components

### 1. Block Generator (`feasible_block_builder.py`)

Generates candidate macro-blocks using randomized DFS/GRASP:

- **MacroBlock**: Fully-valid, pre-sequenced campaign block
- **TemplateGraph**: Directed graph built from template DataFrame
- **generate_candidate_macro_blocks()**: Main entry point

#### Algorithm:
1. Build directed graph from `tpl_df` (orders → edges)
2. For each line, collect seed orders
3. Run randomized DFS walks (forward and backward) from seeds
4. Generate single-order blocks for isolated orders
5. Greedy cluster extension for uncovered orders
6. Sort by cost efficiency (tons/cost)

#### Output:
- `List[MacroBlock]`: Pool of candidate blocks
- `BlockGeneratorStats`: Generation statistics

### 2. Set Packing Master (`joint_master.py`)

Implements the Set Packing CP-SAT model:

#### Variables:
- `x[b]` = 1 if block `b` is selected, 0 otherwise

#### Constraints:
1. **No Overlap**: Each order can appear in at most 1 selected block
   - `sum(x[b] for b containing order i) <= 1` for all orders `i`
2. **Slot Limits**: Per-line slot limit
   - `sum(x[b] for b in line L) <= max_slots` for each line

#### Objective:
- **Maximize** total scheduled tonnage
- `Maximize sum(x[b] * block.total_tons)`

### 3. Direct Sequence Unpacking

Selected blocks are already perfectly sequenced (by construction of the DFS walk). Just unpack `order_ids` directly into `plan_rows`:

```python
for block in selected_blocks:
    for seq_pos, oid in enumerate(block.order_ids, start=1):
        plan_rows.append({
            "order_id": oid,
            "assigned_line": block.line,
            "assigned_slot": campaign_id,
            "master_seq": seq_pos,
            "selected_edge_type": "BLOCK_SEQUENCE",
        })
```

Orders not in any selected block → `dropped_rows` with `drop_reason = "NOT_SELECTED_IN_MACRO_BLOCKS"`

## Configuration

To enable Set Packing Master in your planning configuration:

```python
from aps_cp_sat.config import PlannerConfig

cfg = PlannerConfig()
cfg.model.use_set_packing_master = True  # Enable new Set Packing architecture
```

## Pipeline Integration

The Set Packing architecture is fully integrated with the existing pipeline:

1. **Automatic routing**: When `cfg.model.use_set_packing_master = True`, the `_run_global_joint_model` function automatically delegates to `_run_set_packing_master`

2. **Engine metadata**: The pipeline tracks which architecture is used:
   - `master_architecture`: Either `"set_packing"` or `"legacy_master_local_router"`
   - `used_local_routing`: Always `False` for Set Packing
   - `local_routing_role`: `"set_packing_master"`

3. **Return statistics**: The Set Packing master returns additional statistics:
   - `selected_block_count`: Number of blocks selected
   - `candidate_block_count`: Total candidate blocks generated
   - `block_generation_seconds`: Time spent in block generation
   - `solve_seconds`: Total Set Packing solve time
   - `orders_covered_by_blocks`: Number of unique orders covered
   - `max_walk_depth`: Maximum DFS walk depth achieved

## Usage Example

```python
from aps_cp_sat.config import PlannerConfig
from aps_cp_sat.pipeline import ColdRollingPipeline

# Create config with Set Packing enabled
cfg = PlannerConfig()
cfg.model.use_set_packing_master = True  # Enable new architecture

# Create and run pipeline
pipeline = ColdRollingPipeline()
result = pipeline.run(orders_df, cfg)

# Check engine metadata for Set Packing statistics
if result.engine_meta:
    print(f"Architecture: {result.engine_meta.get('master_architecture')}")
    print(f"Selected blocks: {result.engine_meta.get('selected_block_count')}")
    print(f"Candidate blocks: {result.engine_meta.get('candidate_block_count')}")
```

## Key Benefits

1. **Hard Constraint Guarantees**: Every block is generated by walking valid template edges
2. **No Fallback Edges**: The Local Router's soft penalty fallback is eliminated
3. **Simpler Architecture**: One-phase selection instead of two-phase (assign + sequence)
4. **Better Optimality**: Global optimization over the entire set of valid blocks
